Prework Study Guide

HTML

  • !DOCTYPE html lets the browser know to use html
  • The head element contains information about the webpage.
  • title is the title that will appear in the browser
  • The body element represents the visible content shown to the user.
  • h1, h2, h3, h4, h5, h6 are the tags to use for headlines within the main
  • header is the area at the top of the page
  • footer is the area at the bottom of the page
  • ul means unordered list
  • ol means ordered list
  • li is an item in the list

CSS

  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Link the CSS to the HTML use the following in the head HTML *rel="stylesheet" href="./assets/style.css"*
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • You can control the header and footer with width: 100%; height: 120px; background-color: blue; color: white;
  • Apply CSS rules to images using img
  • Control the ul with CSS
  • Format the text for alignment using p, and size using font-size

Git

  • git init: initialize project to use git
  • git add .: add all changes that you have made
  • git add *filename*: add single file to be saved
  • git commit -m "message": save changes with message
  • git push origin master: push changes to github master
  • git push origin new-branch: push changes to github new-branch
  • git pull origin master: pull changes from github master
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git log: see all previous saved changes
  • git checkout *commit hash*: travel back to old commit
